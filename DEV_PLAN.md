# План разработки приложения "Minimal Gallery" (Android)

## Стек технологий
- **Язык:** Kotlin
- **UI:** Jetpack Compose, Material 3
- **Архитектура:** MVVM
- **DI:** Hilt (или Koin)
- **Асинхронность:** Coroutines + Flow
- **Данные:** Paging 3 (для списка), MediaStore
- **Хранение настроек:** Jetpack DataStore (Preferences)
- **Изображения:** Coil
- **Видео:** ExoPlayer (androidx.media3)
- **Навигация:** Navigation Compose

---

## Этап 1: Инициализация проекта и Архитектура
**Цель:** Подготовить фундамент приложения.

1.  **Создание проекта:**
    -   Минимальный SDK: API 26 (Android 8.0) — баланс охвата и современных фич.
    -   Целевой SDK: API 34/35.
    -   Шаблон: Empty Compose Activity.
2.  **Настройка зависимостей (build.gradle):**
    -   Compose BOM, Navigation, Hilt, DataStore, Paging, Coil, Media3.
3.  **Структура пакетов:**
    -   `ui` (screens, components, theme)
    -   `domain` (models, repository interfaces)
    -   `data` (repositories impl, datastore, mediastore)
    -   `di` (modules)
4.  **Базовая настройка DI:**
    -   Создание Application класса.
    -   Настройка основных модулей Hilt.

---

## Этап 2: Слой данных (Data Layer)
**Цель:** Реализовать получение медиа и хранение настроек.

1.  **Модели данных:**
    -   Создать `MediaItem` (id, uri, mimeType, dateTaken, bucketName/Path).
    -   Создать `AppSettings` (themeConfig, excludedFoldersSet).
2.  **Работа с MediaStore:**
    -   Реализовать `MediaRepository`.
    -   Запрос изображений и видео через `ContentResolver`.
    -   Сортировка по дате (DESC).
    -   Фильтрация по исключенным папкам.
    -   Интеграция с **Paging 3** (создание `Pager` и `PagingSource`).
3.  **Работа с DataStore:**
    -   Создать `SettingsRepository`.
    -   Методы: `getThemeMode()`, `saveThemeMode()`, `getExcludedFolders()`, `toggleFolderExclusion()`.

---

## Этап 3: Основной экран (Галерея)
**Цель:** Отобразить список медиа с высокой производительностью.

1.  **ViewModel:**
    -   `GalleryViewModel`: связь Paging Flow из репозитория и UI State.
2.  **UI Компоненты:**
    -   `LazyVerticalGrid`: сетка с адаптивным количеством колонок.
    -   `MediaCard`: отображение превью (Coil ImageRequest) + иконка типа (видео/фото).
    -   Обработка состояний загрузки и пустого списка.
3.  **Оптимизация:**
    -   Настройка кэширования Coil (memory & disk cache).
    -   Проверка плавности скролла (60 FPS).

---

## Этап 4: Экран просмотра (Viewer)
**Цель:** Полноэкранный просмотр с зумом и видео.

1.  **Навигация:**
    -   Передача `itemId` и текущей позиции для свайпов.
2.  **Просмотр изображений:**
    -   Компонент с поддержкой жестов (zoom, pan). Можно использовать `androidx.compose.foundation.gestures` или готовую обертку.
    -   Свайпы влево/вправо для переключения (используя `HorizontalPager` из Accompanist или Compose Foundation).
3.  **Просмотр видео:**
    -   Интеграция `ExoPlayer` в Compose (`AndroidView` wrapper).
    -   Управление жизненным циклом плеера (pause/resume при уходе с экрана).
4.  **UI:**
    -   Минималистичный интерфейс (скрытие системных баров, появление контролов по тапу).

---

## Этап 5: Настройки и Исключение папок
**Цель:** Реализовать управление темой и фильтрацию контента.

1.  **Экран настроек:**
    -   Выбор темы (Светлая / Темная / Системная) -> обновление `MaterialTheme`.
2.  **Управление папками:**
    -   Запрос уникального списка всех папок (buckets) из MediaStore.
    -   Список с чекбоксами `LazyColumn`.
    -   Логика: при изменении чекбокса -> обновление DataStore -> инвалидация Paging источника (перезагрузка галереи).
3.  **Разрешения (Permissions):**
    -   Запрос `READ_MEDIA_IMAGES` и `READ_MEDIA_VIDEO` при старте (для Android 13+).
    -   Обработка отказа в доступе.

---

## Этап 6: Полировка и Релиз
**Цель:** Устранение багов и подготовка к сборке.

1.  **Тестирование:**
    -   Проверка на разных версиях Android (10, 12, 13, 14).
    -   Проверка поведения при повороте экрана (сохранение состояния плеера/позиции).
2.  **Оптимизация:**
    -   Анализ памяти (Memory Profiler).
    -   Удаление неиспользуемых ресурсов.
3.  **Сборка:**
    -   Подпись релизного APK / AAB.
    -   Иконка приложения (минималистичная).

---

## Критерии приемки MVP
- [ ] Приложение запускается быстрее 1 сек.
- [ ] Скролл галереи плавный даже при 1000+ фото.
- [ ] Видео воспроизводятся со звуком.
- [ ] Исключенные папки мгновенно пропадают из вида.
- [ ] Тема меняется без перезапуска приложения.
- [ ] Нет вылетов при запросе прав доступа.
